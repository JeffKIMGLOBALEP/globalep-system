## 登录会话持久化修复 (第4次 — 修复内联脚本位置)
- 上一次修复的内联脚本放在 loginContainer 和 appContainer 之间，此时 appContainer 尚未解析，导致 `getElementById('appContainer')` 返回 null，无法添加 `active` 类
- 真正修复: 将内联脚本移至所有 HTML 元素之后 (modal 之后、main script 之前)，此时 loginContainer 和 appContainer 均已解析完毕
- **根本原因**: 登录容器默认 `display:flex` 可见，依赖 JavaScript 在 3700+ 行脚本末尾执行 `hideLogin()` 来隐藏。若中途任何代码出错或被阻塞，登录覆盖层会一直显示。
- **解决方案**: 反转默认状态 — 登录容器 HTML 默认 `class="hidden" style="display:none"`
- 新增内联脚本 (line 907-931): 在登录容器 HTML 之后立即运行，检查 localStorage 中 `globalep_user`:
  - 有用户 → 显示 app 容器，保持登录隐藏
  - 无用户 → 显示登录容器
- 移除冗余的 IIFE restoreSession 和 DOMContentLoaded fallback，简化为仅 buildSidebar + showPage
## 登录会话持久化修复 (第6次 — 彻底解决)
- **根因1**: 第815行重复的 .modal-overlay CSS 规则默认 display:flex（应为 none），导致每次加载都显示半透明遮罩
- **根因2**: .login-container CSS 默认 display:flex，模板注入后自动可见
- **修复**:
  1. .login-container → `display:none!important` (CSS 默认隐藏)
  2. .modal-overlay(第815行) → `display:none` (修正重复规则)
  3. 内联脚本: 注入登录后用 inline style `lc.style.display='flex'` 强制显示
  4. restoreApp() 加安全网：已登录时清除残留 loginContainer DOM 节点
  5. doLogout() 注入登录后也强制显示
- 登录改为 template 方式: HTML 中不包含登录容器，通过 `<template>` 按需 clone 注入

## 组织架构页面 "undefined" 修复
- **现象**: 组织架构与业务管理页面只显示 "undefined" 文字
- **原因**: localStorage 中存储了旧的/corrupt 的 performanceData 或 recruitmentData（可能为 null 或非数组），调用 `.filter()` 时 TypeError 导致 renderOrgPage 整个崩溃返回 undefined
- **修复**:
  1. loadFromStorage() 中 JSON.parse 后加 Array.isArray 校验和 try-catch
  2. 无 localStorage 数据时也调用 initPerformanceData() / computeRecruitment()
  3. renderOrgPage() 开头加数据有效性检查
  4. renderOrgPage() 主渲染逻辑包裹 try-catch，出错时显示错误信息和 Reset按钮
